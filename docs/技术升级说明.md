# EduCD-Agent 技术升级说明

## 项目目标

EduCD-Agent 的升级目标是构建“面向智慧教育认知诊断的思维链可解释智能体”。系统不再只依赖规则或自由文本生成，而是以教育数据集、Q 矩阵和 DINA 认知诊断模型为底座，再由 CoT Agent、Report Agent 和 Reflection Agent 生成可解释学情分析。

## 总体架构

```text
学生答题数据 responses.csv
        ↓
题目与知识点 Q 矩阵 q_matrix.csv
        ↓
DINA 认知诊断模型
        ↓
学生知识点掌握度 mastery vector
        ↓
CoT Agent 错因解释与认知断点分析
        ↓
Report Agent 结构化学情报告
        ↓
Reflection Agent 报告自检
        ↓
学生端诊断报告 / 教师端班级分析 / 实验评估
```

## 数据层

数据集位于 `backend/data/`：

- `knowledge_base.json`：知识点库，使用 `C001` 等稳定 `concept_id`。
- `questions.json`：题目文本、标准答案、难度、考查知识点。
- `q_matrix.csv`：题目与知识点的 Q 矩阵。
- `responses.csv`：学生答题记录，包含 `student_id`、`student_name`、`question_id`、`student_answer`、`is_correct`、`score`。

`backend/tools/data_loader.py` 提供统一读取函数和 `validate_dataset()`，用于检查题目、知识点、Q 矩阵和答题记录是否一致。

## DINA 模型层

`backend/models/dina.py` 实现 DINA 训练逻辑，使用 EM 算法估计：

- 学生知识点掌握概率；
- 每道题的 `slip` 参数；
- 每道题的 `guess` 参数；
- 学生最可能掌握模式。

训练入口：

```powershell
cd backend
python train_dina.py
```

训练结果保存到：

- `backend/outputs/mastery_results.json`
- `backend/outputs/dina_params.json`

后端诊断接口只读取训练后的 DINA 输出，不再使用旧的正确率规则回退。

## 智能体层

智能体位于 `backend/agents/`：

- `qwen_client.py`：统一 Qwen 调用入口，优先读取 `DASHSCOPE_API_KEY`。
- `cot_agent.py`：接收 CDM 结构化输入，输出错因、推理链、认知断点和学习建议；同时根据题目 `difficulty` 启用困难度感知动态 CoT，基础题生成 2 步精简链，中等题生成 4 步标准链，复杂题生成 6 步精细链。
- `report_agent.py`：生成结构化学情报告。
- `reflection_agent.py`：检查报告是否与 CDM 一致、是否包含具体错因和可执行建议。

当没有配置 `DASHSCOPE_API_KEY` 时，系统会进入 mock 模式，便于离线演示；配置 Key 后会调用真实 Qwen。

## 后端 API

主要接口包括：

- `GET /api/health`
- `GET /api/students`
- `GET /api/students/detail`
- `GET /api/questions`
- `GET /api/diagnose/<student_id>`
- `GET /api/report/<student_id>`
- `GET /api/full-analysis/<student_id>`
- `GET /api/class/summary`
- `POST /api/train/dina`
- `GET /api/dina/params`
- `GET /api/dataset/validate`

API 返回统一 JSON：

```json
{
  "success": true,
  "message": "请求成功",
  "data": {}
}
```

## 前端展示

前端位于 `frontend/`，基于 Vue、Element Plus 和 ECharts。

学生端展示：

- 学生基本信息；
- DINA 知识点掌握度；
- 薄弱知识点；
- CoT 错因分析；
- 认知断点；
- 结构化学情报告；
- Reflection 自检结果。

教师端展示：

- 班级平均知识点掌握度；
- 薄弱知识点排行；
- 风险学生详情；
- 教师教学建议；
- DINA 训练入口。

## 实验评估

评估脚本位于 `backend/evaluation/`：

- `evaluate_dina.py`：评估 DINA 诊断结果与预设学生类型的一致性。
- `evaluate_reports.py`：比较普通 LLM 报告与 CDM + CoT Agent 报告。

输出结果位于 `backend/outputs/`：

- `dina_evaluation.json`
- `report_evaluation.json`
- `reports.json`
