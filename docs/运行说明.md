# EduCD-Agent 运行说明

本文档说明下载代码后如何在本地运行 EduCD-Agent，包括 DINA 模型训练、后端 API 启动、前端页面启动和常见问题处理。

## 1. 环境要求

建议环境：

- Python 3.10+
- Node.js 20.19+ 或 22.12+
- npm
- Windows PowerShell、CMD、macOS Terminal 或 Linux Shell 均可

项目可以在没有大模型 API Key 的情况下运行。未配置 `DASHSCOPE_API_KEY` 时，系统会自动进入 Mock 模式，适合演示和调试。

## 2. 获取代码

进入项目根目录：

```powershell
cd EduCD-Agent
```

项目结构大致如下：

```text
EduCD-Agent/
├── backend/
├── frontend/
└── docs/
```

## 3. 启动后端

进入后端目录：

```powershell
cd backend
```

创建并激活虚拟环境：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

如果 PowerShell 不允许激活脚本，可以执行：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\.venv\Scripts\Activate.ps1
```

安装依赖：

```powershell
pip install -r requirements.txt
```

首次运行前，先训练 DINA 模型：

```powershell
python train_dina.py
```

训练成功后会生成：

```text
backend/outputs/mastery_results.json
backend/outputs/dina_params.json
```

启动 Flask 后端：

```powershell
python app.py
```

后端默认地址：

```text
http://127.0.0.1:5000
```

可访问健康检查接口：

```text
http://127.0.0.1:5000/api/health
```

## 4. 配置 Qwen API Key

如果只想本地演示，可以跳过本节，系统会使用 Mock 模式。

如果要调用真实 Qwen，在 `backend` 目录下新建 `.env` 文件：

```env
DASHSCOPE_API_KEY=你的DashScope_API_Key
DASHSCOPE_MODEL=qwen3.6-plus
```

也可以在 PowerShell 临时设置：

```powershell
$env:DASHSCOPE_API_KEY="你的DashScope_API_Key"
$env:DASHSCOPE_MODEL="qwen3.6-plus"
python app.py
```

注意：不要把真实 API Key 提交到代码仓库。

## 5. 启动前端

新开一个终端，进入前端目录：

```powershell
cd EduCD-Agent/frontend
```

安装依赖：

```powershell
npm install
```

启动前端开发服务器：

```powershell
npm run dev
```

前端默认地址通常为：

```text
http://127.0.0.1:5173
```

打开浏览器访问该地址，即可进入系统页面。

## 6. 推荐运行顺序

完整启动顺序如下：

```powershell
cd EduCD-Agent/backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python train_dina.py
python app.py
```

然后新开终端：

```powershell
cd EduCD-Agent/frontend
npm install
npm run dev
```

## 7. 页面使用流程

学生端：

1. 进入“学生个体诊断”页面。
2. 选择学生，例如 `S001`。
3. 点击“开始诊断”。
4. 查看知识点掌握度、薄弱知识点、CoT 错因分析、诊断记忆和学情报告。

教师端：

1. 进入“教师看板”页面。
2. 查看班级平均掌握度、薄弱知识点排行、风险学生和教学建议。
3. 如修改了数据，可以点击“训练 DINA”重新生成诊断结果。

## 8. 常用接口

后端常用接口：

```text
GET  /api/health
GET  /api/students
GET  /api/questions
GET  /api/diagnose/<student_id>
GET  /api/report/<student_id>
GET  /api/full-analysis/<student_id>
GET  /api/class/summary
POST /api/train/dina
GET  /api/memory/<student_id>
POST /api/memory/clear
```

示例：

```text
http://127.0.0.1:5000/api/full-analysis/S001
```

## 9. 数据修改与重新训练

教育数据位于：

```text
backend/data/
├── knowledge_base.json
├── questions.json
├── q_matrix.csv
├── responses.csv
└── student_profiles.json
```

如果手动修改题目、Q 矩阵或学生答题记录，需要重新训练 DINA：

```powershell
cd backend
python train_dina.py
```

如果想重新生成一批仿真数据：

```powershell
cd backend
python tools\build_simulated_dataset.py
python train_dina.py
```

## 10. 评估实验

运行 DINA 诊断评估：

```powershell
cd backend
python evaluation\evaluate_dina.py
```

运行报告质量对比评估：

```powershell
python evaluation\evaluate_reports.py
```

评估结果会输出到：

```text
backend/outputs/
```

## 11. 常见问题

### 1. 提示未找到 DINA 掌握度结果

说明还没有训练 DINA。执行：

```powershell
cd backend
python train_dina.py
```

### 2. Qwen 请求超时

真实大模型调用可能耗时较长。可以先不配置 `DASHSCOPE_API_KEY`，使用 Mock 模式完成演示。

### 3. 前端无法访问后端

确认后端已经启动，并且地址为：

```text
http://127.0.0.1:5000
```

前端接口默认会请求该地址。

### 4. 修改数据后页面结果没有变化

需要重新训练 DINA，并刷新前端页面：

```powershell
cd backend
python train_dina.py
python app.py
```

### 5. npm run build 出现 chunk 过大提示

这是 Vite 的构建体积提示，不影响本地运行和演示。

## 12. 最小演示命令

如果已经安装过依赖，可以直接运行：

终端一：

```powershell
cd EduCD-Agent/backend
python train_dina.py
python app.py
```

终端二：

```powershell
cd EduCD-Agent/frontend
npm run dev
```

然后访问：

```text
http://127.0.0.1:5173
```
