# EduCD-Agent 实验结果说明

## 实验目的

本实验用于验证升级后的 EduCD-Agent 是否相比普通文本报告更具有诊断依据和可解释性。重点关注三个问题：

1. DINA 是否能输出符合预设学生画像的知识点掌握度；
2. CDM + CoT Agent 报告是否比普通 LLM 报告更结构化；
3. 加入 CDM 和 CoT 后，报告是否更能体现具体错因、认知断点和可执行建议。

## 实验数据

实验使用 `backend/data/` 下的自构造仿真教育数据集：

- 32 道数学题；
- 12 个知识点；
- 40 名学生；
- 1280 条答题记录；
- 每道题通过 Q 矩阵绑定一个或多个知识点。
- 学生画像覆盖全面掌握型、代数薄弱型、函数薄弱型、几何综合薄弱型、统计概率薄弱型和综合风险型等类型。

## 实验一：DINA 诊断一致性

运行命令：

```powershell
cd backend
python evaluation/evaluate_dina.py
```

输出文件：

```text
backend/outputs/dina_evaluation.json
```

主要指标：

- `type_accuracy`：DINA 预测学生类型与预设画像的一致比例；
- `average_weak_concept_jaccard`：预测薄弱知识点集合与预设薄弱集合的平均 Jaccard 相似度；
- `average_abs_dina_baseline_delta`：DINA 掌握度与简单正确率基线的平均绝对差异。

这些指标用于说明 DINA 不是简单复述正确率，而是结合 Q 矩阵和题目参数进行潜在掌握状态估计。

## 实验二：普通 LLM 报告 vs CDM + CoT Agent 报告

运行命令：

```powershell
cd backend
python evaluation/evaluate_reports.py
```

输出文件：

```text
backend/outputs/report_evaluation.json
backend/outputs/reports.json
```

比较维度：

- `structure`：报告结构完整度；
- `cdm_consistency`：是否与 DINA 掌握度一致；
- `error_specificity`：是否指出具体错因；
- `actionability`：是否包含可执行学习建议；
- `total_score`：加权总分。

普通 LLM 报告只根据整体正确率生成笼统建议；CDM + CoT Agent 报告会显式引用 DINA 掌握度、错题知识点、认知断点和后续练习路径。

## 实验三：消融对比

`report_evaluation.json` 中的 `ablation` 字段包含三种设置：

1. `only_correct_rate`：只用答题正确率；
2. `dina_mastery`：使用 DINA 知识点掌握度；
3. `dina_plus_cot_agent`：使用 DINA + CoT Agent。

预期结论：

- 只用正确率可以描述整体表现，但难以定位具体知识点；
- DINA 可以定位低掌握知识点，但缺少自然语言解释；
- DINA + CoT Agent 可以同时输出知识点弱项、错因解释、认知断点和学习建议。

## 注意事项

评估脚本默认使用 mock 模式生成报告评估，以保证结果可重复、运行速度稳定。项目实际演示或接口调用时，如果环境变量中配置了 `DASHSCOPE_API_KEY`，后端会调用真实 Qwen。
